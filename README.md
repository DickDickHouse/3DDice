# 3DDice
Throwing 3D Dice
想做一個立體投擲骰子的android apps.
前提重點:盡可能模組化，目的是當修改功能或debug時不會干擾或誤觸不必改動的地方引致死bug甚至無法顯示等。所以
i.	要把不相關的class/function/pararmeter設定獨立開來。
ii.	例如埸景空間的設定和顯示相關的class/function/parameters放到一個檔，把骰子的設定和顯示放到另一個檔，把擲骰子的動作和rendering放到另一個檔。把能量bar放到另一個檔。如此類推。
內容
1.	先以模擬座標設定立體空間為2000x2000x2000(x,y,z)，左上角為(0,0,2000)，右下角為(2000,2000,0),黑色底色, 為了容易顯示出立體空間，在畫面的正前方，正左方和底部會有相隔100的中灰色網線，即上述3面看起來就各有20x20即400格子。先以45度角顯示空間。可讓用家雙手指zoom in zoom out。
2.	一粒立體骰子, 先以200x200x200(x,y,z)作體積，顏色為奶白色，要像真正骰子，每面(共6面)都有不同點數（圓點），即分別正中央1紅圓點，2個圓點，3個圓點，4個圓點，5個點及6個點
3.	骰子預設放在模擬座標空間的(1000,1000,1600) 位置
4.	能量bar, 橫向，整體尺寸為1000(長),50(粗)，黑色底，外框為5厚，銀色，儲能量的能量條為螢光綠色，100%滿能量時的總尺寸為990(長),40(粗)。能量bar左上角座標先放在模擬座標的(500,1600,2000)位置

5.	模擬物理特性: 即像擲骰子動作
a.	先單手指按著不放,便會儲能量，能量bar的能量條(螢光綠色部份)由0%向右增加到100%，如果手指沒放開，能量會跳回0%再開始直到手指鬆開，
b.	在按著時候(儲能量)，手指可同時向任何方向快速滑動，操作就像模擬真正擲骰子動作，當鬆開手指時，骰子就會按照鬆開手前所儲的能量並方向配合速度擲出去, (所以這裏可能要記錄鬆手前一刻的座標，然後鬆手後的一刻座標並計算角度和速度，以所儲的能量，模擬把骰子擲出。
c.	然後骰子摸摸物理公式的向著那個方向力度的拋物線擲出並翻滾，到達地面(先設定骰子的中心座標跟地面z=0的距離為骰子半徑即50時為著地)會有彈起和翻滾。
d.	以先設定的模擬座標的四邊為牆，連同地面，約果骰子撞到4邊或地面會有反彈效果。
e.	因為有反彈效果，便會需要有衰減，否則永遠彈來彈去不會停，先設定系數可以調校衰減幅度，容後看效果再調整。這裏先設定各反彈造成30%的速度和力度的衰減。
f.	骰子要顯示立體面的圓點，也要案3D效果顯示。
g.	當骰子停下來，向上的一面便是骰子結果。點擊畫面任何位置後骰子回到模擬座標空間的(1000,1000,1600) 位置。
